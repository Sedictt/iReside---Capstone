# UI/UX Review Workflow

Use this workflow to evaluate a component or screen against industry-standard UI/UX principles.

## Steps

1. **Invoke UI/UX Standards Enforcer**
   - Read the instructions in [UI/UX Standards Enforcer](file:///c:/Users/JV/Documents/GitHub/iReside/.agent/skills/ui_ux_standards/SKILL.md)
   - Use the "Required Review Framework" to analyze the target file(s)

2. **Analyze the Screen/Component**
   - Identify the primary user goal
   - Evaluate clarity, structure, and consistency
   - Check accessibility (contrast, labels, keyboard use)
   - Verify responsiveness

3. **Generate Review**
   - Use the "Response Format" defined in the skill:
     - Goal
     - What works
     - Issues
     - Recommended improvements
     - Accessibility / responsiveness notes
     - Final assessment

4. **Propose Changes**
   - If acceptable, offer to apply the recommended improvements to the codebase
