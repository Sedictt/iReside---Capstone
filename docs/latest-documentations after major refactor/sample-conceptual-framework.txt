This study's conceptual framework is based on the Input-Process-Output (IPO) model, which gives a structured method to analyze the iReside platform's development and operational flows. This approach demonstrates how combining modern software design, stakeholder needs, and iterative development cycles resulted in an effective property and tenant services platform for Barangay Marulas. The system remains flexible by leveraging a continuous feedback loop, guaranteeing that the end result adequately handles the practical issues of local rental management.
The Input phase of the framework includes the fundamental elements required to begin research. This starts with a comprehensive knowledge base gathered from related literature and existing systems to identify technological gaps in current property management solutions. Technically, the input contains a complex technological stack consisting of Next.js and React.js for the frontend, TypeScript for type-safe development, and Supabase for secure PostgreSQL database management. Furthermore, the integration of the Gemini 1.5 Flash API provides important intelligent input to automation. These software tools are backed up by hardware resources, such as high-performance laptops for development and smartphones for user testing, all of which are hosted on Vercel cloud servers. Standard TCP/IP protocols are used over Wi-Fi and mobile data to provide seamless real-time synchronization.
The Process phase focuses on the transformation of these inputs through the Evolutionary Prototyping Software Development Life Cycle (SDLC). Unlike linear models, this iterative process begins by identifying core property and rental needs specific to the residential landscapes of Barangays Canumay, Maysan, Malinta, and Marulas within Valenzuela City. Following the initial requirement gathering, the process moves into the design of user interfaces and the structuring of database schemas tailored to these urban contexts. The development stage involves active coding and the seamless integration of AI modules, which is immediately followed by a rigorous user validation phase. During this stage, direct feedback is gathered from a diverse group of landlords and tenants across the four identified barangays to identify necessary functional adjustments. This leads to a systematic refinement stage where the platform is updated based on actual user experience, ensuring that the final delivery is technically robust, culturally relevant, and user-centric. This cycle repeats iteratively until the platform fully meets the established project requirements and is deemed ready for full-scale deployment across the target locales.
This study's primary output is the iReside: Integrated Property and Tenant Services Platform, a cross-platform application that can be accessed via web or mobile devices. Aside from the program, the outputs contain extensive user manuals with "How-To" assistance for both landlords and tenants, ensuring high acceptance rates. In addition, detailed technical documentation is created, defining the system architecture and API standards for future maintenance and scaling. Finally, a Feedback system is built within the framework, allowing for ongoing study of operational data and user ideas. This ensures that the system may be refined even after it has been deployed, keeping it relevant and efficient in the long-term administration of rental properties.
