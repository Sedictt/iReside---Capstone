# Conceptual Framework

This study's conceptual framework is based on the Input-Process-Output (IPO) model, which provides a structured method to analyze the development and operational flows of the iReside platform. This approach demonstrates how combining modern software engineering, stakeholder needs, and iterative development cycles resulted in an exclusive, private property and tenant management ecosystem for the multi-unit rental market within Valenzuela City. iReside is designed as a landlord-centric management platform where each property functions as an isolated, private ecosystem — tenants are provisioned exclusively through approved onboarding workflows, and there is no public listing, unit searching, or marketplace discovery. The system remains flexible by leveraging a continuous feedback loop, ensuring that the end result adequately addresses the practical challenges of local rental management within this private-access architecture.

## Input

The Input phase of the framework encompasses the fundamental elements required to initiate the research and development effort. This begins with a comprehensive knowledge base gathered from related literature and existing systems to identify technological gaps in current property management solutions, particularly those serving the multi-unit residential rental market in Barangays Marulas, Canumay, Malinta, and Maysan within Valenzuela City.

Technically, the input comprises a modern full-stack architecture: **Next.js 16 (App Router)** with **TypeScript** for the application framework, **React 19** with **Tailwind CSS 4**, **Radix UI**, and **Framer Motion** for a polished, responsive frontend, and **Supabase** for secure PostgreSQL database management, real-time subscriptions, authentication, and cloud storage. Specialized libraries include **@dnd-kit/core** for the drag-and-drop spatial canvas and **Geist + Rethink Sans** for typographic consistency. The integration of the **Groq Llama 3.1 8B** API via the OpenAI SDK provides the intelligent layer powering the iRis AI Assistant, landlord analytics, message moderation, and maintenance triage through Retrieval-Augmented Generation (RAG). These software tools are supported by hardware resources such as high-performance laptops for development and smartphones for responsive design testing, with the application hosted on cloud infrastructure through Supabase. Standard TCP/IP protocols are used over Wi-Fi and mobile data to provide seamless real-time synchronization across all user roles.

Additionally, domain-specific inputs inform the system's design: the private-access model (where properties operate as isolated ecosystems analogous to private servers), the three-role permission structure (Super Admin, Landlord, Tenant), and the walk-in application workflow that gates all tenant provisioning through landlord approval — reflecting the real-world onboarding dynamics of the target market.

## Process

The Process phase focuses on the transformation of these inputs through the **Evolutionary Prototyping Software Development Life Cycle (SDLC)**. Unlike linear models, this iterative process begins by identifying core property and rental management needs specific to the residential landscapes of Barangays Canumay, Maysan, Malinta, and Marulas within Valenzuela City — anchored around the platform's private, landlord-controlled ecosystem model.

Following the initial requirement gathering, the process moves into the design of three distinct user interfaces — the **Landlord Portal** for comprehensive property, tenant, and financial management; the **Tenant Portal** for residency tools, communication, and AI-assisted support; and the **Super Administrator Portal** for platform governance and system monitoring — along with the structuring of database schemas with Row-Level Security (RLS) to enforce strict data isolation between roles.

The development stage involves active coding and the integration of core system modules: the **2D Spatial Canvas** with modular floor planner for visual property management, the **walk-in application and lease finalization workflows** for tenant onboarding, the **enhanced digital lease signing system** with dual and remote signing modes, the **real-time financial ledger** for invoice and payment tracking, the **maintenance request management pipeline**, the **real-time messaging engine** with AI-powered content moderation, the **iRis AI Assistant** with RAG capabilities, the **Community Hub** for building-wide engagement, and the **AI-driven landlord analytics dashboard** with reporting and export tools.

This is immediately followed by a rigorous user validation phase, where direct feedback is gathered from a diverse group of landlords and tenants across the four identified barangays to identify necessary functional adjustments. This leads to a systematic refinement stage where the platform is updated based on actual user experience, ensuring that the final delivery is technically robust, culturally relevant, and user-centric. This cycle repeats iteratively until the platform fully meets the established project requirements and is deemed ready for deployment across the target locales.

## Output

The primary output of this study is **iReside: Integrated Property and Tenant Management Platform**, a unified web-based application delivered through both desktop-optimized dashboards and a mobile-optimized Progressive Web App (PWA) interface accessible via any modern web browser. The platform comprises three role-specific portals:

- **Landlord Portal** — featuring the modular floor planner, walk-in application processing, lease management with enhanced digital signing, real-time financial ledger, maintenance dashboard, AI-driven analytics with PDF/CSV reporting, real-time messaging, and community management tools.
- **Tenant Portal** — providing digital lease signing, a document vault, lease overview with renewal eligibility, a read-only unit map with transfer request capability, maintenance request submission, payment history, real-time messaging with the landlord, the iRis AI Assistant, community engagement, and interactive product tours for onboarding.
- **Super Administrator Portal** — offering live enterprise metrics, registration pipeline monitoring, advanced applicant review with document inspection, user governance with role-specific filtering, and internal administrative notes.

Beyond the application itself, the outputs include comprehensive user documentation with role-specific guides for landlords, tenants, and administrators, ensuring high adoption rates. Detailed technical documentation is produced, defining the system architecture, API standards, database schemas, and security policies for future maintenance and scaling. Finally, a feedback mechanism is built into the framework, allowing for continuous analysis of operational data and user input. This ensures that the system can be refined even after deployment, keeping it relevant and efficient in the long-term administration of rental properties within the private ecosystem model.
