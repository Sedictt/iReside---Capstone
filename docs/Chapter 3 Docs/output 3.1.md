## 3.1 Development Model

This section presents the software development life cycle model used in developing iReside. The proponents adopted the **Evolutionary Model** because the system required continuous refinement as the scope, workflows, and user needs became clearer throughout development.

### Evolutionary Development Model

Figure X. Evolutionary Development Model

The Evolutionary Model is a software development approach in which the system is built and improved through a series of working versions. Instead of attempting to complete the entire system in a single pass, the development team begins with the most essential features, evaluates the output, and then enhances the system in succeeding iterations. This model is appropriate for projects whose requirements may expand, change, or become more detailed as the developers gain deeper understanding of the problem domain and user expectations.

The model was well suited for iReside because the system evolved from a general housing-related concept into a more focused private property management platform for landlords, tenants, and administrators. As the project progressed, the proponents refined the system boundaries, clarified role-based workflows, and expanded modules according to the actual operational needs of residential property management. Through the evolutionary approach, each working build served as a basis for evaluation, adjustment, and feature enhancement.

In the early stage of development, the proponents identified the core functions that were necessary for the first usable version of iReside. These included authentication, role-based access, property and unit management, lease handling, and tenant account provisioning. Establishing these core modules first made it possible to create the system foundation before more advanced functions were added.

After the initial version was established, the system was improved incrementally through repeated development cycles. Additional modules such as maintenance request management, real-time messaging, document vault access, community features, analytics dashboards, and the iRis AI assistant were integrated in later iterations. Each cycle involved reviewing the previously developed version, identifying missing needs or weak areas, and implementing improvements based on the system’s intended users and actual workflow requirements.

The evolutionary model also supported interface and process refinement. Since iReside has multiple user roles, namely administrator, landlord, and tenant, each iteration allowed the proponents to improve the usability of dashboards, navigation flow, data presentation, and role-specific actions. This made it easier to adjust forms, pages, and transaction processes such as lease signing, maintenance tracking, billing review, and communication tools without disrupting the entire system architecture.

Testing and validation were likewise performed throughout the development process. Every new version of the system was checked to ensure that newly added functions worked properly and remained consistent with previously implemented features. This continuous verification helped the proponents detect issues early, maintain data integrity, and preserve secure access control across the different modules of the system.

In the context of iReside, the Evolutionary Model followed a practical sequence. First, the proponents gathered and analyzed the fundamental requirements of the system. Second, an initial working version was designed and implemented with the essential modules. Third, the version was reviewed to determine which features needed refinement or expansion. Fourth, new capabilities were added in succeeding iterations, such as landlord analytics, tenant self-service tools, community interaction, and AI-assisted support. This process continued until the system reached a more complete and stable version that addressed the project objectives.

Overall, the Evolutionary Model was the most appropriate development model for iReside because it allowed the system to grow gradually while remaining functional at every stage. It enabled the proponents to respond to changing requirements, improve role-specific workflows, and build the platform in manageable increments. Through this approach, iReside was developed as a more adaptive, practical, and user-centered property management system.
